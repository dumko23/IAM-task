Project task

Делаем имитацию системы управления пользователями в очень сокращенном виде.
Все действия должны быть без перезагрузки страницы. Все через Ajax
(Bootstrap 4.x, jQuery, Ajax, PHP).
Основа шаблона доступна по ссылке:
https://drive.google.com/file/d/1N5mruHA_f_LO1mFBDVVTGJwT0LwKD97D/view

1. у нас должна быть одна страница
2. выводим таблицу с пользователями (детали ниже)
4. выводим над и под таблицей блок со следующими моментами:
	a. кнопка add
	b. селектбокс c первым пунктом-названием Please Select, котрый имеет следующие функциональные элементы (1. Set active, 2. Set not active, 3. Delete)
	c. кнопку OK

*Таблица

1. поле чекбокс (логика работы стандартная - нажал и все выбрались, отжал какой-то то один и отжался верхний общий, так как не все выбраны). Это чек бокс для групповых действий
2. Name (First Name Last Name)
3. Status (выводим зеленый кружок, если актив, и серый, если нет)
4. Role (admin/user)
5. Options состоит с двух кнопок
	а. Edit (иконки)
	b. Delete (иконки)

*форма добавления/правок (модальное окно бутстрап)
должно быть ОДНО окно на добавление и правку
состоит со следующих полей
1. First Name (текст)
2. Last Name (текст)
2. Status (чекер как у гугла, полоска с кружечком, https://assets.materialup.com/uploads/a6456f3a-defc-4cb4-8754-9a7054ba63d4/codepen_-_pure_css3__scss__google_material_design_toggle_switch.jpg)
3. Role (селект бокс)

*модальные окна (не должно быть системных конфирмов):
- подтверждение удаления юзера перед удалением (конфирм-окно)
- окно предупреждения, что не выбраны юзеры, а в селектбоксе выбрана опция и нажата кнопка «ОК»
- окно предупреждения, что выбран юзер, нажата кнопка «ОК», но не выбрано действие в селектбоксе

*бекенд
тут PHP и, конечно же, mysql

*развернуть где-нибудь для проверки, скинуть гит линк для проверки кода
- скинуть линк на развернутый проект
- скинуть линк на гит

- валидация на беке обязательна на не пустые поля, на фронте по желанию
- методы бека должны отвечать в таком видеи формате
Примеры:
{status: false, error:{code: 100, message: "not found user"}}

{status: true, error:null, id: 1}
